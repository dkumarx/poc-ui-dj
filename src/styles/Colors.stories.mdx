# Color Palette

POC UI Core provides a `Color` object that defines the HEX color codes used in
the POC UI style guide. `Color` groups the POC UI palette into these basic
groups:
- `Color.Primary`.  
  Primary brand colors.
- `Color.Background`  
  Colors used for block background fills.
- `Color.Neutral`  
  Blacks, greys and other neutral colors.
- `Color.Button`  
  Colors for button states.

To use `Color` in your application, import it as follows.

```
import { Color } from 'poc-ui-mat-lib';

// or

import Color from 'poc-ui-mat-lib/styles/Color';
```

A number of convenience types are also available. These are useful if you would
like to constrain color values passed to functions or assigned to variables, for
example. The color types are:
- `PrimaryColorType`  
  `Color.Primary` color codes. 
- `BackgroundColorType`  
  `Color.Primary` color codes.
- `NeutralColorType`  
  `Color.Primary` color codes.
- `ButtonColorType`  
  `Color.Primary` color codes.
- `ColorType`  
  All color codes.

The code snippet below shows how `ButtonColorType` could be used to ensure that
a function that toggles the color of a button only receives valid color codes.
```
import { ButtonColorType, Color } from 'poc-ui-mat-lib';

function toggleButtonColor(activeColor: ButtonColorType) {
  // ... logic goes here ...
}

// Will work
toggleButtonColor(Color.Button.Pressed);

// Won't work
toggleButtonColor(Color.Primary.Plum);
```
